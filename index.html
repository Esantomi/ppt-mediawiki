위키와 미디어위키

  • 위키(Wiki)란?
    • 위키(wiki)는 불특정 다수가 협업을 통해 직접 내용과 구조를 수정할 수 있는 웹사이트
    • 일반적인 위키에서 텍스트는 단순화된 마크업 언어(위키 마크업)을 이용하여 작성
    • 공동체용 웹사이트나 조직 내 인트라넷 등 다양한 용도로 이용

출처 : 『』(2010), p

위키와 미디어위키

  • 위키위키웹(WikiWikiWeb)
    • 워드 커닝햄(Ward Cunningham)이 만든 최초의 위키 소프트웨어
      • “위키란 동작하는 가장 단순한 온라인 데이터베이스” (커닝햄의 위키 정의)
    • “위키”는 “빠른”을 뜻하는 하와이어 “wiki”에서 따옴

출처 : 『』(2010), p

위키와 미디어위키

  • 위키의 특징
    • 위키의 가장 핵심적인 개념은 사이트를 방문한 누구나 위키 웹사이트 내의 문서를 고치거나 새로 만들 수 있다는 것
      • 기본적인 웹브라우저만 있으면 되며, 추가적인 확장 기능을 요구하지 않음
    • 위키는 서로 다른 문서들을 단순히 링크하는 것만으로도 의미 있는 주제 간의 연계를 만들어 내고 해당 문서가 존재하는지 아닌지까지도 보여 줄 수 있음

출처 : 『』(2010), p

위키와 미디어위키

  • 미디어위키란?
    • 미디어위키(MediaWiki)는 자유 웹 기반 위키 소프트웨어이며, PHP와 MySQL을 사용하고 GNU GPL 라이선스로 배포됨
    • 미디어위키는 현재 위키백과와 같은 위키미디어 재단의 여러 위키에 이용되며, 다른 백과사전 형식의 위키에서도 많이 이용됨

출처 : 『』(2010), p

AWS 서버에 미디어위키 압축 풀기

  • 본 내용은 이전 강의를 통해 AWS 라이트세일 인스턴스를 생성한 이후에 따라갈 수 있으므로, 미비된 경우 이전 강의를 참고 바람
  • 인스턴스 생성을 모두 마쳤다면 AWS 홈페이지에 접속하여 먼저 로그인을 한 뒤, 아래 슬라이드를 참고하여 진행 바람

1. aws 홈페이지 접속 후 생성한 인스턴스 접속

2. SSH를 사용하여 연결 버튼을 눌러 웹 SSH 열기

복사+붙여넣기

  1. SSH에 상기 명령어를 차례로 입력 

							cd htdocs
							wget https://releases.wikimedia.org/mediawiki/1.38/mediawiki-1.38.4.tar.gz
							tar -xvzf mediawiki-1.38.4.tar.gz
							mv mediawiki-1.38.4 wiki

타자로 치지 말고 위에서부터 한 줄씩
마우스 우클릭+붙여넣기

  • 명령어 설명(참고)
    • cd htdocs : htdocs 폴더로 경로 이동
    • wget : 주소에 있는 파일 다운로드
    • tar : 압축 파일의 압축을 푸는 명령
    • mv 폴더1 폴더2 : 폴더1의 내용물을 폴더2라는 폴더를 만들어 안으로 옮기기

4. 설정해 둔 고정 IP로 접속하면 상기 화면이 나옴

데이터베이스 및 사용자 생성

  • LAMP를 선택해 미디어위키 설치 준비는 완료했음
  • 하지만 데이터베이스 및 데이터베이스 ID 생성은 직접!
    • 데이터를 저장할 공간과 그 공간을 사용할 사람의 ID를 만드는 과정

정신 차리고 복사+붙여넣기만 잘하면 된다!

  1. SSH에 상기 명령어를 차례로 입력

							cd ..
							cat bitnami_application_password
							mysql -u root -p
							CREATE DATABASE wikidb;
							SHOW DATABASES;
							CREATE USER wiki@localhost IDENTIFIED BY 'e52fwUUfE9hyoi6r1';
							GRANT ALL PRIVILEGES ON wikidb.* TO wiki@localhost WITH GRANT OPTION;

이전 슬라이드의 실제 예시를 꼭 참고 및 비교하며 진행!!

  • 명령어 설명(참고)
    • cat bitnami_application_password : 기본으로 주어지는 mysql 비밀번호를 출력
    • mysql -u root -p : mysql 접속
      • cat bitnami_application_password를 쳐서 나온 비밀번호를 입력(비밀번호는 눌러도 눈에 안 보임)
    • CREATE DATABASE 이름; : 이름이라는 데이터베이스 생성
    • CREATE USER ID@localhost IDENTIFIED BY 'PW'; : ID, PW로 아이디 생성
    • GRANT ALL PRIVILEGES ON 데이터베이스1.* TO ID@localhost : 데이터베이스1에 대해 모든 권한 부여

11. 상기 이미지 참고하여 체크 및 체크 해제

LocalSettings.php 서버에 올리기

  • LocalSettings.php : 데이터베이스 정보를 포함한 위키와 관련된 대부분의 설정 관리
    • 설치 시 저절로 생성(안 될 시 수동으로 다운로드)되며, 사용자가 직접 wiki 폴더 안에 넣어 줘야 함
  • 서버에 어떻게 파일을 올릴 수 있을까?🤔🤔
  • FTP/SFTP 프로그램인 FileZilla를 활용!
    • FTP(File Transfer Protocol) : 서버에 파일을 올리는 프로그램
    • SFTP(Secure FTP) : 서버에 안전하게 파일을 올리는 프로그램
